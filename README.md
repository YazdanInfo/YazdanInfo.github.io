YazdanInfo.github.io

A personal website template powered by GitHub Pages – fork it, customize it, and launch your own site in minutes.

---

About

This repository contains the source code for my personal website (YazdanInfo.github.io). It is designed to be a simple, ready‑to‑use template for anyone who wants to create their own GitHub Pages site without starting from scratch.

---

Features

· 🚀 Zero configuration – just fork and go.
· 🎨 Fully customizable – replace index.html, add your own CSS and JavaScript.
· 🌐 GitHub Pages ready – automatically builds and hosts your site.
· 🔗 Custom domain support – link your own domain via the Pages settings.

---

Getting Started

1. Fork this repository

Click the Fork button at the top right of this page to create a copy under your GitHub account.

2. Rename the repository (optional)

If you want your site to be available at yourusername.github.io, rename the forked repository to yourusername.github.io (replace yourusername with your GitHub username).

3. Customize your site

· Replace index.html with your own HTML file.
· Add your stylesheets (.css) and scripts (.js) in the root folder.
· Update any other assets (images, fonts, etc.) as needed.

4. Enable GitHub Pages

· Go to your repository Settings → Pages.
· Under Branch, select main (or master) and /root.
· Click Save – your site will be published at https://yourusername.github.io (or the custom domain you set).

5. Set a custom domain (optional)

In the Pages settings, enter your custom domain in the Custom domain field and save. Don't forget to configure your DNS records with your provider.

---

Customization Tips

· HTML – modify index.html to change content and structure.
· CSS – add your own styles (e.g., style.css) and link it in the <head>.
· JavaScript – add scripts (e.g., script.js) at the end of the <body>.
· All files placed in the root of the repository will be served directly.

---

Deployment

GitHub Pages deploys automatically on every push to the selected branch. No extra build steps are required – just commit and push your changes.
